# US-005: Feedback visual apos exclusao bem-sucedida

**Feature:** FEAT-002
**Status:** Backlog
**Estimativa:** P
**Criado em:** 2026-06-06

---

## User Story
**Como** medico logado
**Quero** ver uma confirmacao visual quando um paciente for excluido com sucesso
**Para** ter certeza de que a acao foi concluida

## Criterios de Aceitacao

```gherkin
Cenario 1: Toast de sucesso apos exclusao
  DADO que confirmei a exclusao de um paciente
  QUANDO a exclusao e processada com sucesso
  ENTAO vejo um toast/notificacao "Paciente excluido com sucesso" por 3 segundos

Cenario 2: Mensagem de erro se exclusao falhar
  DADO que confirmei a exclusao de um paciente
  QUANDO o backend retorna erro (500, rede cai)
  ENTAO vejo uma mensagem de erro clara e o paciente permanece na lista
```

## Notas Tecnicas

- Reutilizar o sistema de notificacao/toast ja existente no frontend (se houver) ou implementar um simples com `setTimeout`
- Nao usar `alert()` nativo do browser

## Dependencias

- US-004: endpoint de exclusao precisa existir primeiro

## Criterios de Rejeicao

- Usar `alert()` nativo
- Nao mostrar nenhum feedback apos a acao
- Manter o spinner de loading indefinidamente se a requisicao falhar
