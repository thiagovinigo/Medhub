# US-006: Abrir caso salvo em modal/pagina de leitura

**Feature:** FEAT-003
**Status:** Backlog
**Estimativa:** G
**Criado em:** 2026-06-06

---

## User Story
**Como** medico logado
**Quero** abrir um caso clinico salvo e ver seu laudo completo
**Para** consultar a analise de um caso anterior sem precisar re-analisar

## Criterios de Aceitacao

```gherkin
Cenario 1: Abrir caso salvo
  DADO que clico em um caso clinico no historico
  QUANDO o modal/pagina de detalhe abre
  ENTAO vejo o titulo do caso, data da analise, especialidade, queixa principal e o laudo completo

Cenario 2: Conteudo do laudo exibido por secoes
  DADO que estou na view de detalhe de um caso
  ENTAO as secoes do laudo estao organizadas nas mesmas abas do CaseWizard (Analise, Correlacao, Profissional, Leigo)

Cenario 3: Dados do paciente exibidos
  DADO que o caso tem um paciente vinculado
  QUANDO abro o detalhe
  ENTAO vejo o nome do paciente, data de nascimento e sexo no cabecalho

Cenario 4: Imagens nao disponiveis
  DADO que as imagens originais dos exames nao sao armazenadas
  QUANDO abro o detalhe
  ENTAO vejo uma nota informando que as imagens originais nao estao disponiveis, apenas o laudo textual
```

## Notas Tecnicas

- Criar componente `CaseReportView` separado do `CaseWizard` — nao reutilizar o wizard para leitura
- Consumir `GET /api/cases/{id}` (verificar se endpoint existe; se nao, criar)
- Parsear `exams_summary` (JSON string) para exibir lista de exames do caso
- Parsear `analysis` e `research` para as abas correspondentes

## Dependencias

- US-003 (FEAT-001): o clique que dispara a abertura desta view

## Criterios de Rejeicao

- Permitir edicao dos campos em modo leitura
- Exibir dados de um caso de outro usuario
- Crashar se `exams_summary` for JSON invalido
