# US-025: Schema do banco atualizado automaticamente no deploy sem perda de dados

**Feature:** FEAT-010
**Status:** Backlog
**Estimativa:** G
**Criado em:** 2026-06-06

---

## User Story
**Como** desenvolvedor
**Quero** que mudancas de schema do banco sejam aplicadas automaticamente no deploy
**Para** nao precisar executar SQL manualmente no Supabase a cada alteracao de modelo

## Criterios de Aceitacao

```gherkin
Cenario 1: Migracao aplicada no startup
  DADO que existe uma nova migracao Alembic pendente
  QUANDO o container do Railway faz restart (novo deploy)
  ENTAO a migracao e aplicada antes do servidor comecar a servir requisicoes

Cenario 2: Schema inalterado nao causa erro
  DADO que nao ha migracoes pendentes
  QUANDO o servidor reinicia
  ENTAO o startup ocorre sem erros relacionados ao banco

Cenario 3: Migracao falhou — rollback
  DADO que uma migracao com erro foi deployada
  QUANDO a migracao falha no startup
  ENTAO o processo encerra com codigo de saida nao-zero (Railway mantem a versao anterior rodando)

Cenario 4: Baseline do schema atual gerado corretamente
  DADO que o banco em producao ja existe com tabelas criadas pelo create_all
  QUANDO o Alembic e inicializado pela primeira vez
  ENTAO o baseline e gerado sem tentar recriar as tabelas existentes
```

## Notas Tecnicas

- Instalar: `alembic`, configurar `alembic.ini` e `env.py` apontando para `DATABASE_URL`
- Gerar revisao inicial: `alembic revision --autogenerate -m "baseline"`
- Marcar producao: `alembic stamp head` (executar uma vez manualmente no Supabase)
- Script de startup do Railway: `alembic upgrade head && uvicorn main:app --host 0.0.0.0 --port 8000`
- Remover o `create_all()` do `startup()` em `main.py` apos Alembic configurado

## Dependencias

- Nenhuma dependencia de outras user stories — e infraestrutura base.

## Criterios de Rejeicao

- Manter `create_all()` em paralelo com Alembic (causa conflito de estado)
- Migracao que apaga dados existentes ao ser aplicada
- Migracao que nao funciona em SQLite (ambiente de dev local)
