# US-019: Baixar PDF de um caso ja salvo a partir do historico

**Feature:** FEAT-007
**Status:** Backlog
**Estimativa:** P
**Criado em:** 2026-06-06

---

## User Story
**Como** medico logado
**Quero** baixar o PDF de um caso salvo diretamente do historico ou da view de detalhe
**Para** nao precisar re-abrir o wizard ou re-analisar o caso

## Criterios de Aceitacao

```gherkin
Cenario 1: Botao de PDF na view de detalhe do caso
  DADO que estou na view de detalhe de um caso salvo (FEAT-003)
  ENTAO vejo um botao "Baixar PDF"

Cenario 2: Download correto do PDF
  DADO que clico em "Baixar PDF" na view de detalhe
  ENTAO o PDF gerado contem os dados do caso salvo (nao um PDF vazio)
```

## Notas Tecnicas

- Reutilizar o endpoint `POST /api/pdf/case` criado em US-018
- Os dados do caso ja foram carregados para renderizar a view — reutilizar os mesmos dados para o POST

## Dependencias

- US-018: endpoint `/api/pdf/case` precisa existir
- US-006: view de detalhe do caso (FEAT-003)

## Criterios de Rejeicao

- Fazer uma segunda requisicao GET ao backend para buscar os dados do caso — os dados ja estao carregados no frontend
- Abrir PDF em nova aba sem fazer download
