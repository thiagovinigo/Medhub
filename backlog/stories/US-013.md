# US-013: Filtrar historico por especialidade

**Feature:** FEAT-005
**Status:** Backlog
**Estimativa:** P
**Criado em:** 2026-06-06

---

## User Story
**Como** medico logado
**Quero** filtrar meu historico por especialidade medica
**Para** encontrar rapidamente todos os casos de neurologia ou cardiologia que analisei

## Criterios de Aceitacao

```gherkin
Cenario 1: Filtro por especialidade
  DADO que estou no historico unificado
  QUANDO seleciono "Neurologia" no filtro de especialidade
  ENTAO apenas registros de neurologia sao exibidos

Cenario 2: Limpar filtro
  DADO que apliquei um filtro de especialidade
  QUANDO clico em "Limpar filtros"
  ENTAO todos os registros voltam a ser exibidos

Cenario 3: Nenhum resultado para o filtro
  DADO que seleciono uma especialidade sem registros
  ENTAO vejo a mensagem "Nenhum registro encontrado para esta especialidade"
```

## Notas Tecnicas

- Filtro pode ser client-side se o numero de registros for pequeno (<200)
- Para conjuntos maiores, adicionar query param `?specialty=neuro` no endpoint de timeline
- Quick analyses tem `modality` mas nao `specialty`; considerar incluir apenas casos clinicos neste filtro

## Dependencias

- US-012: timeline unificada precisa existir

## Criterios de Rejeicao

- Nao mostrar mensagem quando o filtro retorna zero resultados
- Filtro que nao responde imediatamente (latencia >300ms para filtro client-side)
