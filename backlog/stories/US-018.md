# US-018: Gerar PDF completo de um caso clinico com todas as secoes

**Feature:** FEAT-007
**Status:** Backlog
**Estimativa:** G
**Criado em:** 2026-06-06

---

## User Story
**Como** medico logado
**Quero** gerar um PDF completo de um caso clinico com todas as secoes do laudo
**Para** imprimir ou arquivar o laudo de forma profissional

## Criterios de Aceitacao

```gherkin
Cenario 1: PDF com todas as secoes
  DADO que estou no resultado de um caso clinico no CaseWizard
  QUANDO clico em "Gerar PDF"
  ENTAO o PDF contem: cabecalho com titulo do caso e data, dados do paciente, queixa principal, lista de exames, analise detalhada, correlacao clinica, laudo profissional, laudo leigo, sugestoes e referencias

Cenario 2: PDF com formatacao legivel
  DADO que gero o PDF
  ENTAO o documento tem fonte legivel (minimo 10pt), margens adequadas e secoes claramente separadas por titulos

Cenario 3: PDF sem dados de paciente se caso e anonimo
  DADO que o caso nao tem paciente vinculado
  QUANDO gero o PDF
  ENTAO a secao de dados do paciente e omitida sem erros
```

## Notas Tecnicas

- Criar novo endpoint `POST /api/pdf/case` com schema proprio:
  `{ title, patient_context, chief_complaint, exams_summary, analysis, research, metadata }`
- Nao alterar o endpoint existente `POST /api/pdf` (usado pelo Quick Analysis)
- No `pdf_generator.py`, criar funcao `generate_case_pdf()` separada de `generate_report_pdf()`

## Dependencias

- Nenhuma dependencia de outras user stories — e independente de FEAT-003.

## Criterios de Rejeicao

- Gerar PDF sem as secoes de correlacao e especialidade (que sao o diferencial do caso clinico)
- Alterar o comportamento do endpoint `/api/pdf` existente
- Crashar se alguma secao do laudo estiver vazia
