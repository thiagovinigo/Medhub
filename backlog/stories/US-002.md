# US-002: Distinguir quick analysis de caso clinico no historico

**Feature:** FEAT-001
**Status:** Backlog
**Estimativa:** P
**Criado em:** 2026-06-06

---

## User Story
**Como** medico logado
**Quero** ver claramente qual registro e uma analise rapida e qual e um caso clinico completo
**Para** saber o que esperar ao clicar em cada item do historico

## Criterios de Aceitacao

```gherkin
Cenario 1: Identificacao visual de quick analysis
  DADO que tenho registros de ambos os tipos no historico
  QUANDO vejo a lista
  ENTAO itens de quick analysis tem badge "Quick" ou icone de raio-x simples

Cenario 2: Identificacao visual de caso clinico
  DADO que tenho registros de ambos os tipos no historico
  QUANDO vejo a lista
  ENTAO itens de caso clinico tem badge "Caso Clinico" com a especialidade (ex: "Caso Clinico - Neuro")

Cenario 3: Informacoes complementares por tipo
  DADO que vejo um caso clinico na lista
  ENTAO consigo ver o numero de exames do caso (ex: "3 exames") e o titulo do caso
```

## Notas Tecnicas

- Badge pode ser implementado com CSS simples, sem biblioteca adicional
- A especialidade esta disponivel no campo `title` do `ClinicalCase` ou pode ser inferida do `exams_summary`

## Dependencias

- US-001: a lista de casos precisa existir antes de adicionar diferenciacao visual

## Criterios de Rejeicao

- Usar apenas cor como unico diferenciador (viola acessibilidade)
- Nao exibir nenhuma informacao contextual alem da data
