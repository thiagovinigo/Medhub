# US-024: Analise do mesmo arquivo nao reprocessa se resultado ja esta em cache

**Feature:** FEAT-009
**Status:** Backlog
**Estimativa:** G
**Criado em:** 2026-06-06

---

## User Story
**Como** medico
**Quero** que o sistema identifique se ja analisei um arquivo identico anteriormente
**Para** receber o resultado instantaneamente sem gastar tokens da API Groq

## Criterios de Aceitacao

```gherkin
Cenario 1: Cache hit — arquivo ja analisado
  DADO que fiz upload do mesmo arquivo JPEG anteriormente (mesmo hash SHA-256)
  QUANDO faco upload novamente
  ENTAO recebo o resultado da analise anterior em menos de 1 segundo (sem chamar a API Groq)

Cenario 2: Cache miss — arquivo novo
  DADO que faco upload de um arquivo que nunca foi analisado antes
  ENTAO o sistema processa normalmente chamando a API Groq

Cenario 3: Cache isolado por usuario
  DADO que usuario A analisou um arquivo
  QUANDO usuario B envia o mesmo arquivo
  ENTAO usuario B tambem obtem o resultado em cache (o cache de analise nao e sensivel a identidade do usuario — apenas ao conteudo)
```

## Notas Tecnicas

- Calcular `sha256(file_bytes)` antes de salvar o arquivo temporario
- Buscar em `exam_records` por `file_hash` (adicionar coluna `file_hash VARCHAR`)
- Se encontrado e `analysis` nao e nulo, retornar o registro existente
- Requer migracao de schema (adicionar coluna `file_hash`) — depende de FEAT-010 para producao segura

## Dependencias

- FEAT-010: adicionar coluna `file_hash` requer migracao Alembic

## Criterios de Rejeicao

- Cache compartilhado entre usuarios para dados medicos sensiveis (nao se aplica — hash de arquivo nao e dado pessoal; apenas o resultado e armazenado com user_id correto)
- Cache que nao expira nunca (considerar TTL de 30 dias ou baseado em storage disponivel)
