# US-021: Ver erro explicativo se o limite de requisicoes for atingido

**Feature:** FEAT-008
**Status:** Backlog
**Estimativa:** P
**Criado em:** 2026-06-06

---

## User Story
**Como** medico usando a plataforma
**Quero** receber uma mensagem clara quando atingir o limite de requisicoes
**Para** entender o que aconteceu e saber quando posso tentar novamente

## Criterios de Aceitacao

```gherkin
Cenario 1: Mensagem de rate limit
  DADO que fiz muitas requisicoes em pouco tempo
  QUANDO o backend retorna HTTP 429
  ENTAO vejo a mensagem: "Limite de requisicoes atingido. Aguarde alguns segundos e tente novamente."

Cenario 2: Botao de retry apos rate limit
  DADO que recebi o erro de rate limit
  ENTAO vejo um botao "Tentar novamente" que fica habilitado apos 10 segundos

Cenario 3: Rate limit configurado por endpoint
  DADO que o rate limit esta configurado no backend via slowapi
  ENTAO o endpoint /api/analyze aceita no maximo 10 requisicoes por minuto por IP
  E o endpoint /api/auth/login aceita no maximo 5 requisicoes por minuto por IP
```

## Notas Tecnicas

- Backend: instalar `slowapi` e configurar `Limiter` com `key_func=get_remote_address`
- Retornar header `Retry-After` no 429 para o frontend saber quando reativar o botao
- Frontend: ler o header `Retry-After` e iniciar countdown para o botao de retry

## Dependencias

- Nenhuma dependencia de outras user stories.

## Criterios de Rejeicao

- Mostrar o mesmo erro generico de rede para rate limit (usuario nao entende o que aconteceu)
- Nao ter o botao de retry — usuario precisa recarregar a pagina
