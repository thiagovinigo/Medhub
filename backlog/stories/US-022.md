# US-022: Nao conseguir submeter o CaseWizard sem ao menos um arquivo carregado

**Feature:** FEAT-008
**Status:** Backlog
**Estimativa:** P
**Criado em:** 2026-06-06

---

## User Story
**Como** medico usando o CaseWizard
**Quero** ser impedido de submeter o formulario sem arquivos carregados
**Para** nao gerar um caso vazio ou receber um erro confuso do backend

## Criterios de Aceitacao

```gherkin
Cenario 1: Botao de submit desabilitado sem arquivos
  DADO que abri o CaseWizard mas nao carreguei nenhum arquivo
  ENTAO o botao "Analisar" esta desabilitado ou cinza

Cenario 2: Mensagem de validacao ao tentar submeter vazio
  DADO que o botao de submit esta habilitado mas nenhum exame tem arquivo
  QUANDO clico em "Analisar"
  ENTAO vejo a mensagem de validacao: "Adicione pelo menos um arquivo de exame para continuar"

Cenario 3: Submit habilitado com pelo menos um arquivo
  DADO que carreguei pelo menos um arquivo em qualquer exame do wizard
  ENTAO o botao "Analisar" esta habilitado
```

## Notas Tecnicas

- Validacao deve ser client-side (nao depender de resposta do backend para erros basicos)
- Verificar que ao menos um `exam.files.length > 0` antes de habilitar o submit
- O backend ja retorna 422 para casos sem arquivos — mas a validacao frontend evita a requisicao desnecessaria

## Dependencias

- Nenhuma dependencia de outras user stories.

## Criterios de Rejeicao

- Validar apenas no backend (causa request desnecessario e latencia)
- Bloquear o submit sem explicar o motivo ao usuario
