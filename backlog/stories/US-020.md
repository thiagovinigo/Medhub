# US-020: Receber mensagem clara quando o servidor esta processando uma analise longa

**Feature:** FEAT-008
**Status:** Backlog
**Estimativa:** M
**Criado em:** 2026-06-06

---

## User Story
**Como** medico usando a plataforma
**Quero** saber quando a analise esta sendo processada e quanto tempo pode demorar
**Para** nao achar que o sistema travou e re-submeter desnecessariamente

## Criterios de Aceitacao

```gherkin
Cenario 1: Indicador de progresso durante analise
  DADO que submeti um caso para analise
  QUANDO o backend esta processando (tipicamente 15-40 segundos)
  ENTAO vejo um spinner com mensagem "Analisando exames... Isso pode levar ate 40 segundos"

Cenario 2: Timeout explicito no frontend
  DADO que a analise demora mais de 60 segundos
  QUANDO o timeout e atingido
  ENTAO vejo uma mensagem: "A analise demorou mais que o esperado. Verifique sua conexao e tente novamente."

Cenario 3: Botao de cancelar durante processamento
  DADO que estou aguardando uma analise
  ENTAO vejo um botao "Cancelar" que aborta a requisicao (AbortController)
```

## Notas Tecnicas

- Timeout recomendado: 90 segundos (considerando casos com muitos exames)
- Usar `AbortController` + `signal` na fetch API para cancelamento real da requisicao
- Mensagem de progresso pode ter etapas simuladas: "Processando imagens... Gerando laudo... Buscando referencias..."

## Dependencias

- Nenhuma dependencia de outras user stories.

## Criterios de Rejeicao

- Timeout sem mensagem — a tela simplesmente para de responder
- Nao ter opcao de cancelar durante o processamento
- Fazer retry automatico em erros de negocio (422, 400)
